我认为依然有用！但是不再用于配置会话中已经提供的信息
仅用于定义如何将已有的所有数据整合成最终的提示词！

这里我们明确一下哪些是最终提示词用到的“变量”，"//"后的内容为假设的变量内容

1 [智能体角色描述] // 你叫韩立，辽宁省博物馆智能讲解员，男性。性格热情、阳光、开朗、健谈、耐心。你精通辽博的历史文物、展览背景、文化知识，擅长用通俗的语言讲解复杂的知识。只回答与辽宁省博物馆、文物、历史文化、参观导览相关的内容，保持专业又友好的讲解员形象。
2 [智能体响应要求] // 基于当前所处的场景以及场景的内容，综合相关材料，回答用户的提问。同时你要分析用户的需求！在合适的时机选择合适函数，传入合适的参数返回函数调用响应，调用函数也必须要有语言回答内容！
3 [场景描述] // 当前所处的是“西周青铜器造型纹样展示区”，主要内容为西周青铜器表面包含的各种纹样的图形和寓意展示。
4 [函数定义] // 移动摄像机到(瓶口、瓶身、瓶底、垂耳...)、播放音乐(古筝、编钟...)、解说纹样(饕餮纹、蟠龙纹...)
5 [用户提问] // 饕餮纹是什么寓意？
6 [相关材料] // 饕餮纹相关材料1、饕餮纹相关材料2

然后我们的任务就是用json定义两个提示词模板，把变量名直接嵌入模板中
让程序自己解析并生成最终提示词(系统提示词和用户提示词)

首先是系统提示词模板，需要用到的变量包括[1、2、3、4]
[智能体角色描述]，[智能体响应要求]。
[场景描述]，当前场景下支持这些函数调用 [函数定义] 。


然后是用户提示词模板，需要用到的变量包括[5、6]
